================================================================================
Wonder Cabinet is distributed as a podcast on Apple Podcasts, Spotify,
YouTube Music, and via RSS to any other podcast app. Each platform has
specific image requirements. The current Show_Cover-Wonder-Cabinet.png
(item [13] in Tier 3) covers the basic submission spec, but we'd like
the full kit refreshed.


[F] SHOW COVER ART — the primary podcast artwork
    Format   : PNG or JPG, RGB, no alpha
    Size     : 3000 x 3000 px (square, 1:1)
    Min spec : 1400 x 1400 (Apple); 3000 x 3000 strongly preferred
    Notes    : This is the single most important podcast asset — it's
               what listeners see in their podcast app's library, search
               results, and now-playing screen. Must be legible at
               55 x 55 px (the smallest size Apple displays it). Avoid
               small text. No "explicit" / "new episode" badges (Apple
               rejects these).
               Replaces the current Show_Cover-Wonder-Cabinet.png.

    Format   : PSD / Figma component + sample PNG export

    Notes    : A template that producers can drop episode-specific
               imagery / titles into for each new episode's artwork.
               Should leave a clear "title zone" and use consistent
               brand framing (border, color block, etc.) so episodes
               feel like a series.

================================================================================
APPENDIX — BEHIND THE SCENES  (optional reading)
================================================================================
Skip this unless you're curious about why the directory has so many files.

The site runs on Ghost (a CMS). Some of the images in this folder are
served directly to browsers — those are the SVGs and PNGs in Tier 1 and
Tier 4 that say "auto-generated" or "PhotoSwipe."

Other images in this folder (the wordmark, logo, galaxy bg, social-share
PNG) live here as DESIGN MASTERS but the live site actually loads its
copy from Ghost's CDN, because we uploaded them through the Ghost admin
panel. So when you deliver a new master:

  • For the SVGs in Tier 1, items [3], [4], [6], [7], plus all of Tier 2:
    we drop the new file into this folder and redeploy the theme.
  • For the wordmark [1], galaxy bg [2], logo [5], OG image [12]:
    we replace the file here AND re-upload the new version through Ghost
    admin so production picks it up.
  • For the show cover [13] and the divider [14]: same as the SVGs —
    drop in folder, redeploy.
  • The favicon set is currently orphaned, so we don't have to do anything
    with it unless we decide to revive it.

Either way, the designer's deliverable is the same: the master files at
the sizes specified above. We handle the upload paths.
